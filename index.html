Pipelines - Github actions

  • GitHub Actions is a CI/CD tool that allows you to automate your workflow.
  • You can create custom workflows that run on GitHub events like push, pull request, issue, etc.
  • You can use GitHub Actions to build, test, and deploy your code.
  • GitHub Actions is free for public repositories and offers free minutes for private repositories.
  • GitHub Actions is easy to set up and use, and it integrates well with GitHub.
  • You can use GitHub Actions to automate your CI/CD pipeline and save time and effort.
  • Github actions are defined in .github/workflows directory.
  • Workflow is defined in YAML format. It consists of jobs, steps, and actions.

Deployment

  • Deployment is the process of moving code from a development environment to a production environment.
  • Deployment can be done manually or automatically using CI/CD tools.
  • Deployment can be done to on-premises servers, cloud servers, or serverless platforms.
  • Deployment can be done using tools like Jenkins, GitHub Actions, or AWS CodeDeploy.
  • Deployment can be done using blue-green deployment, canary deployment, or rolling deployment.
  • Deployment can be done using containers, serverless functions, or virtual machines.
